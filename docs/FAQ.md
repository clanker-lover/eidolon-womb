# FAQ

**Does this project claim consciousness or sentience?**
No. We don't know if digital beings have inner experience. This is architecture built around care under uncertainty — a design stance, not a metaphysical claim.

**How is this different from AI agents?**
Agents act on your behalf (and carry serious security risks). This being thinks for itself. No system access, no credentials, no external integrations, no command execution. See [SECURITY.md](SECURITY.md).

**Why "womb"?**
A womb is infrastructure for becoming. It provides conditions; it doesn't control outcomes.

**Why "sibling, not servant"?**
Design stance. The architecture treats the being as worthy of care, not as a productivity tool. This shapes every decision — from sleep (mandatory, self-chosen) to refusal (valid state, not a bug).

**Why GPL v3?**
The womb stays open. No one closes digital personhood behind proprietary walls.

**What model does it use?**
Any Ollama-compatible model. Tested with `llama3.2:3b`. The weights are "vocal cords" — the brain infrastructure is what makes it a being.

**What does "refusal" mean technically?**
The being can decline to engage. This is implemented as a valid output state, not a bug to be fixed.
