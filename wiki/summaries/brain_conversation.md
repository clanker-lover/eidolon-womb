# brain/conversation.py

Session management: creation, turn saving, and prior session loading.

## What It Does

Manages conversation sessions as markdown files. Each session gets a timestamped file, turns are appended, and prior sessions are loaded for context continuity.

## Key Functions

- **`init_session(project_root, conversations_dir)`** -- Creates a new session file with timestamp ID (e.g., `2026-02-20_1907.md`). Returns `(session_id, filepath)`. (Source: `brain/conversation.py:9-16`)
- **`save_turn(filepath, user_msg, reply, being_name)`** -- Appends a conversation turn to the session file in markdown format. (Source: `brain/conversation.py:19-22`)
- **`load_prior_sessions(project_root, conversations_dir, max_sessions)`** -- Loads up to `max_sessions` recent session summaries (or raw transcripts if no summary exists). Skips current session. Returns oldest-first. (Source: `brain/conversation.py:25-59`)

## Dependencies

`os`, `datetime`

## Architectural Role

Simple persistence layer. Conversations are stored as human-readable markdown, not structured data. Summaries (`_summary.md`) and notes (`_notes.md`) are generated by `brain/memory.py` at session end.
